# Block 3: Claude Code 연결 = Vault 규칙서

## EXPLAIN

> **전제:** Block 1에서 만든 **Vault 폴더**가 있어야 한다(루트에 CLAUDE.md를 두므로). Block 2의 폴더 골격까지 있으면 "vault 지도"를 더 정확히 적을 수 있다.

### 도입 — CLAUDE.md 3줄 요약

- **CLAUDE.md** = 프로젝트(=Vault) 루트에 두는 마크다운 파일
- **매 세션 시작 시 Claude가 자동으로 읽는다**
- 즉 매번 "이렇게 일해줘"를 반복하지 않아도 되는 **작업 규칙서**

> 🔁 CLAUDE.md를 깊게 배우고 싶다면 **Lv.2 Session 1**에서 자세히 다룬다. 여기서는 *Vault용 규칙서*로 핵심만 본다.

### Vault용 CLAUDE.md에 들어갈 것

Vault용 CLAUDE.md에는 이런 항목을 적는다:

- **문체** — 한국어, 존댓말 등
- **폴더 규칙** — 새 노트는 어디에, frontmatter 포함
- **수정 금지** — `.obsidian/` 폴더는 절대 건드리지 않기
- **연결 규칙** — 관련 개념은 `[[wikilink]]`로
- **안전 규칙** — API key·토큰·개인정보 읽거나 저장하지 않기

### vault 지도 한 단락 — 가장 큰 효율 레버

CLAUDE.md에 **"이 vault는 이렇게 생겼다"**(폴더 구성 + frontmatter 표준)를 한 단락 적어두면, Claude가 폴더를 추론하지 않고 **읽고 바로 시작**한다. 구조 효율의 핵심은 폴더 분류법이 아니라, 구조를 AI가 읽는 곳에 적어두는 것이다.

### 안전 순서 — 처음부터 쓰기 권한을 주지 않는다

```
분석 → 보고 → 특정 파일 수정 → diff 검토 → 반영
```

- **Plan Mode(읽기 전용)로 시작**한다
- 첫 요청은 항상 **"수정 말고 보고만 해줘"**
- vault 전체 대량 수정 전에는 계획부터 받는다

이 순서가 "되돌리기 어려운 사고"를 막는다.

## EXECUTE

다음을 직접 실행해보세요:

**Step 1: Vault용 CLAUDE.md 만들기**
- Obsidian Vault 폴더를 Claude Code(IDE / 터미널 / 데스크톱 Code 탭)에서 연다
- `templates/claude-md-template.md`를 참고해 vault 루트에 `CLAUDE.md` 작성
- 최소 항목: 문체 / 폴더 규칙 / `.obsidian` 수정 금지 / vault 구조 한 단락

**Step 2: 첫 요청은 "보고만"**
- Plan Mode로 전환 후 입력:
  ```
  이 vault 구조를 읽고, 두 번째 뇌로 쓰기에 부족한 점을 보고해줘. 아직 파일은 수정하지 마.
  ```
- Claude의 분석 보고를 읽어본다

> ✅ **이렇게 되면 성공:** vault 루트에 `CLAUDE.md`가 생기고, Plan Mode에서 "보고만 해줘"라고 했을 때 Claude가 **파일을 고치지 않고** 분석 보고만 돌려주면 성공.

## QUIZ

**문제:** Claude에게 Vault를 처음 맡길 때 가장 안전한 첫 요청은?

| 선택지 | 내용 |
|--------|------|
| A | "vault 전체를 알아서 정리해줘" |
| B | "필요 없는 파일은 다 지워줘" |
| C | "구조를 읽고 보고만 해줘 (수정하지 마)" |

**정답:** C — 분석→보고→diff→반영 순서가 안전하다. Plan Mode(읽기 전용)로 시작해 보고를 먼저 받는다. A는 "알아서 전체 정리"가 대량 수정이라 되돌리기 어렵고, B는 삭제까지 맡겨 사고 위험이 가장 크다.

---

**적용 문제 (시나리오):** CLAUDE.md를 막 만들고 Claude에게 vault를 처음 맡긴다. 노트가 어수선해서 빨리 정리하고 싶다. 가장 적절한 첫 한 수는?

| 선택지 | 내용 |
|--------|------|
| A | "지금 바로 모든 노트의 frontmatter를 통일하고 폴더도 재배치해줘"라고 한 번에 실행시킨다 |
| B | Plan Mode에서 "구조와 문제점을 보고만 해줘"로 시작해, 보고를 본 뒤 특정 파일부터 diff로 검토하며 반영한다 |
| C | `.obsidian/` 설정까지 포함해 vault 전체를 Claude가 알아서 최적화하게 둔다 |

**정답:** B — 첫 요청은 항상 "보고만", 그다음 특정 파일을 diff로 검토하며 좁게 반영하는 게 안전 순서다. A는 검토 없는 대량 수정이라 되돌리기 어렵고, C는 건드리면 앱이 깨질 수 있는 `.obsidian/`까지 맡겨 가장 위험하다.
