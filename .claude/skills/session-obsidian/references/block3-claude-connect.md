# Block 3: Claude Code 연결 = Vault 규칙서

## EXPLAIN

### 도입 — CLAUDE.md 3줄 요약

- **CLAUDE.md** = 프로젝트(=Vault) 루트에 두는 마크다운 파일
- **매 세션 시작 시 Claude가 자동으로 읽는다**
- 즉 매번 "이렇게 일해줘"를 반복하지 않아도 되는 **작업 규칙서**

> 🔁 CLAUDE.md를 깊게 배우고 싶다면 **Lv.2 Session 1**에서 자세히 다룬다. 여기서는 *Vault용 규칙서*로 핵심만 본다.

### Vault용 CLAUDE.md에 들어갈 것

세팅북의 AGENTS.md 예시를 Vault에 맞게 옮기면 이렇다:

- **문체** — 한국어, 존댓말 등
- **폴더 규칙** — 새 노트는 어디에, frontmatter 포함
- **수정 금지** — `.obsidian/` 폴더는 절대 건드리지 않기
- **연결 규칙** — 관련 개념은 `[[wikilink]]`로
- **안전 규칙** — API key·토큰·개인정보 읽거나 저장하지 않기

### vault 지도 한 단락 — 가장 큰 효율 레버

CLAUDE.md에 **"이 vault는 이렇게 생겼다"**(폴더 구성 + frontmatter 표준)를 한 단락 적어두면, Claude가 폴더를 추론하지 않고 **읽고 바로 시작**한다. 구조 효율의 핵심은 폴더 분류법이 아니라, 구조를 AI가 읽는 곳에 적어두는 것이다.

### 안전 순서 — 처음부터 쓰기 권한을 주지 않는다

```
분석 → 보고 → 특정 파일 수정 → diff 검토 → 반영
```

- **Plan Mode(읽기 전용)로 시작**한다
- 첫 요청은 항상 **"수정 말고 보고만 해줘"**
- vault 전체 대량 수정 전에는 계획부터 받는다

이 순서가 "되돌리기 어려운 사고"를 막는다.


## QUIZ

**문제:** Claude에게 Vault를 처음 맡길 때 가장 안전한 첫 요청은?

| 선택지 | 내용 |
|--------|------|
| A | "vault 전체를 알아서 정리해줘" |
| B | "필요 없는 파일은 다 지워줘" |
| C | "구조를 읽고 보고만 해줘 (수정하지 마)" |

**정답:** C — 분석→보고→diff→반영 순서가 안전하다. 처음부터 대량 수정·삭제 권한을 주면 되돌리기 어렵다. Plan Mode(읽기 전용)로 시작해 보고를 먼저 받는다.
