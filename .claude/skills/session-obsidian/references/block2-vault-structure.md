# Block 2: Vault 구조 — PARA에서 더 나아가기

## EXPLAIN

### 1. 가장 널리 쓰이는 출발점 — PARA

PARA는 모든 자료를 **실행 가능성(actionability)** 축으로 4개로 나눈다.

| 폴더 | 뜻 | 예 |
|------|-----|-----|
| **P**rojects | 마감 있는 진행 작업 | "6월 워크샵 기획" |
| **A**reas | 지속 관리 영역 | "팀 운영", "건강" |
| **R**esources | 참고 자료 | "디자인 레퍼런스" |
| **A**rchive | 끝난 것 | 완료 프로젝트 |

널리 쓰이는 좋은 출발점이다. **하지만 정답은 아니다.**

### 2. PARA의 한계 (AI 협업 관점)

- **분류 마찰** — "이건 Resource야 Note야 Draft야?" 매번 고민 → vault가 썩는 1순위 원인
- **경로 이동** — Project가 끝나면 Archive로 이동 → `[[wikilink]]`·참조가 깨짐
- **결정적:** AI는 **폴더 트리를 뒤지지 않는다.** Claude는 frontmatter를 grep하고, 링크를 따라가고, 검색한다. 폴더 깊이는 AI에게 약한 신호다.

### 3. 원리 — 구조의 3층: 폴더·파일명·frontmatter

| 층위 | 역할 | 예 |
|------|------|-----|
| **폴더** | 거친 필터(생애주기) | 받은함 / 활성 / 참고 / 보관 |
| **파일명** | 식별·정렬·링크 | 개념=제목 그대로 / 자료=날짜 prefix |
| **frontmatter** | 정밀 필터 | `type` / `status` / `tags` / `project` |

→ **폴더 깊이보다 파일명·frontmatter 일관성에 투자하라.** 그리고 구조 설명서는 CLAUDE.md에 적어 AI가 추론 대신 읽게 하라.

### 4. 진화 예시 3종 (더 나은 방법들)

> 맛만 보고 넘어가도 된다 — "PARA 말고도 길이 많다"는 감각만 얻으면 충분.

| 방식 | 분류 축 | 특징 |
|------|---------|------|
| **Johnny.Decimal** | 고정 번지(예 `11.01`) | 주소가 명확해 AI에 좋음, 단 규율이 빡셈 |
| **Flat + 메타데이터** | 거의 안 나눔, 검색·frontmatter | AI 최적이나 사람이 길 잃기 쉬움 |
| **생애주기 3티어** | 활성/참고/보관 | 균형점. 예: `00_보관` / `01_활성` / `99_장기축적` (번호=순서 아닌 위계) |

### 5. 파일명 규칙 — 노트 종류별로 (옵시디언 친화)

옵시디언에선 **파일명 = 노트 제목 = `[[링크]]` 대상**이다. 그래서 노트 종류에 따라 다르게 쓴다:

| 노트 종류 | 파일명 방식 | 예 |
|------|---------|-----|
| 개념·영구 노트 | 사람이 읽는 **제목 그대로** | `두 번째 뇌` → `[[두 번째 뇌]]` (링크 예쁨) |
| 자료성(클리핑·일지·리포트) | **날짜 prefix** `YYYY-MM-DD_제목` | `2026-06-09_LLM-Wiki-요약` |
| 살아있는 산출물(기획·문서) | 날짜 + **버전**(선택) | `2026-06-09_제안서_v1.2.0` |

공통: **공백 대신 `-`/`_`**(CLI·링크 안정), 한 vault 안에서 **일관성** 유지.

> **사례:** 어떤 팀은 운영 문서엔 `YYYY-MM-DD_제목_v1.0.0`, 한 번 받고 안 바꾸는 아카이브엔 `제목_YYMMDD`처럼 타입별 규칙을 둔다. **내 규칙을 정했으면 CLAUDE.md에 한 줄 적어** Claude가 새 노트를 그대로 따르게 하라.

### 6. 핵심 메시지

> **정답 폴더는 없다.** PARA는 출발점일 뿐, 내 일에 맞게 **더 나은 구조를 스스로 발견·변형**하라.
>
> 핵심 한 줄: *"남의 시스템을 그대로 쓰지 말고, 내 시스템에 맞게 변형하라."*

## QUIZ

**문제:** AI 협업 관점에서 폴더 구조를 설계할 때 가장 중요한 원칙은?

| 선택지 | 내용 |
|--------|------|
| A | 폴더는 얕게(생애주기), 정밀 분류는 frontmatter로 |
| B | 폴더를 최대한 깊고 세밀하게 나눈다 |
| C | PARA 구조를 그대로 외워서 쓴다 |

**정답:** A — AI는 깊은 폴더 트리를 탐색하지 않고 frontmatter를 grep하고 링크를 따라간다. 폴더는 거친 생애주기 필터로 얕게 두고, 정밀 분류는 메타데이터에 맡긴 뒤 구조 설명은 CLAUDE.md에 적는 게 가장 효율적이다. 정답 구조는 없으니 스스로 변형하라.
