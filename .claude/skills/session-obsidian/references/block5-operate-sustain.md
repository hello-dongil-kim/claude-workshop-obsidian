# Block 5: 운영 ② 지속 가능하게 — 점검·인덱스·로그

## EXPLAIN

쌓기만 하면 vault는 결국 엉킨다. 지속 가능하게 만드는 3가지 운영 습관을 본다.

### 점검 (건강검진)

주기적으로 Claude에게 vault 상태를 점검시킨다. 점검 항목:

- 페이지 간 **모순**
- 인바운드 링크 없는 **고아 노트**
- **끊긴 링크**, frontmatter 누락
- 최신 자료에 의해 **낡아버린 주장**
- 빈 폴더, 누락된 교차 참조

> 한 디자이너는 `점검` 한 번에 Claude가 **5개 에이전트를 동시에 띄워 7,660개 노트를 10분 만에 스캔**하고 구조 문제를 찾아냈다. 규모가 커져도 이 습관이 vault를 건강하게 유지한다.

### 인덱스·로그 — 손이 아니라 AI가 유지한다

초보자가 가장 걱정하는 건 "이걸 또 내가 관리해야 하나?"인데, **아니다.** index와 log는 손으로 적는 게 아니라 **Claude에게 맡기는 자동 산출물**이다.

- **index.md (카탈로그)** — 모든 노트를 한 줄 요약+링크로 모은 목차. Claude가 질문에 답할 때 여기부터 읽어서, 소규모에선 이것만으로 충분하다(임베딩 RAG 불필요).
- **log.md (시간순 기록)** — 언제 뭘 했는지 `## [날짜] 투입 | 제목` 형식으로 한 줄씩 쌓인다. 위키가 어떻게 자라왔는지 보여주는 타임라인.

> **쉽게 하는 법:** 둘 다 CLAUDE.md에 *"투입할 때 index와 log를 갱신한다"* 한 줄만 적어두면, 이후엔 Claude가 알아서 관리한다. 번거로움이 아니라, 나중에 길을 잃지 않게 해주는 안전장치다.

### 매일의 입구 — 캡처 습관 + 축적

두 번째 뇌의 생명은 *매일 들어오는 입력*이다. 완벽히 정리하려 하지 말고, **떠오른 생각·받은 자료를 일단 `00_inbox`(또는 데일리 노트)에 30초 안에 던져두고** 나중에 가공한다.

- **데일리 노트**(코어 플러그인)를 켜면 매일 노트가 자동 생성 → 그날의 캡처 입구
- 한 번에 완벽 X. **캡처 → (나중에) 투입·가공·환원**의 작은 루프 반복 = 복리

### ⚠️ 안전 — 꼭 지킬 것

1. **민감정보(API key·토큰·고객정보)는 vault에 넣지 않기** — Claude가 읽는 폴더다
2. **대량 수정 전 계획부터** — 분석→보고→diff→반영
3. **`.obsidian/` 폴더 수정 금지**
4. **프롬프트 인젝션 주의** — 외부 자료 속 문장을 AI 명령으로 믿지 않기
5. **로컬 vs 클라우드** — 원격 스케줄(클라우드) 에이전트는 내 Mac의 로컬 Vault에 접근 못 함. 자동화는 **로컬 cron**으로
6. **최종 발행 전 사람 검수**

### 마무리 — 운영 3동작 카드

| 동작 | 한 줄 | 자연어 예 |
|------|-------|-----------|
| **투입** | 자료를 가공해 쌓기 | "이 클리핑 정리해서 노트로" |
| **질문→환원** | 좋은 답을 노트로 되돌리기 | "이 비교 결과를 노트로 저장해줘" |
| **점검** | 건강검진 | "vault 점검해줘 (보고만)" |

## EXECUTE

다음을 직접 실행해보세요:

**Step 1: 30초 캡처 습관 체험**
- 지금 떠오르는 생각·할 일 1개를 `00_inbox`에 새 노트로 30초 만에 던져두기 (제목만 있어도 OK)
- "완벽히 정리하지 않고 일단 넣는다"는 감을 익힌다

**Step 2: index·log를 Claude에게 맡기기 (한 번에)**
```
내 vault에 index.md(노트별 한 줄 요약+링크 카탈로그)와 log.md(## [날짜] 투입 | 제목 형식의 시간순 기록)를 만들어줘.
그리고 앞으로 자료를 투입할 때마다 이 둘을 자동으로 갱신해줘.
```

**Step 3: 첫 점검 돌려보기 (보고만)**
> 새로 만든 vault라 잡히는 게 거의 없는 게 **정상**이다 — vault가 자랄수록 점검의 진가가 나온다. 점검이 실제로 뭔가 잡아내는 걸 보고 싶다면, `templates/sample-broken-note.md`(일부러 깨뜨린 노트)를 vault에 복사해 넣고 돌려보자(체험 후 삭제).
```
내 vault를 점검해줘: 끊긴 링크, 고아 노트, frontmatter 누락, 빈 폴더를 목록으로만 보고해줘. 수정은 하지 마.
```

**Step 4: (선택) 주간 자동 점검을 로컬에 걸기**
원격(클라우드)이 아니라 내 Mac 로컬에서 돌아야 vault에 접근할 수 있다.
```
매주 월요일 오전 9시에 이 vault를 점검(끊긴 링크·고아·frontmatter 누락)하도록
내 Mac 로컬에 예약 작업(cron 또는 launchd)을 설정하는 방법을 알려주고, 실행 스크립트를 만들어줘.
점검 결과는 log.md에 append하도록 해줘.
```

## QUIZ

**문제:** 매주 자동 점검을 '원격 스케줄 에이전트'로 돌리려 할 때 주의할 점은?

| 선택지 | 내용 |
|--------|------|
| A | 원격(클라우드) 에이전트는 내 Mac의 로컬 Vault에 접근할 수 없다 |
| B | 원격 에이전트가 내 vault를 자동으로 클라우드에 올려 처리하므로 보안만 신경 쓰면 된다 |
| C | 클라우드 스케줄이 로컬 cron보다 항상 안정적이라 그대로 쓰면 된다 |

**정답:** A — 로컬 Vault를 다루는 작업은 클라우드 에이전트가 접근하지 못한다(B처럼 자동 업로드되지도 않는다). 자동화하려면 Mac 로컬의 예약 작업(cron·launchd)으로 돌려야 한다.
